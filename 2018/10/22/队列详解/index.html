队列

一种操作受限的线性表，只允许在队头删除元素，队尾插入元素，特点：先进先出

顺序队列

基本思想：分配一块连续的存储单元存放队列中的元素，并设下头指针front：指向队首元素的前一个元素，尾指针rear:指向队尾元素

也可以front指向队首元素，rear指向队尾元素的下一个位置，但是在初始化，入队出队需要相应的变化

节点定义：

初始化： front == -1， rear == -1

ps:如果front指向队首，rear指向队尾的下一个位置，初始化：front == 0, rear == 0

入队： 如果队列满则不能入队(队满：rear == MAXSIZE-1)，否则 ++rear, data[rear] = 入队元素

ps:如果front指向队首，rear指向队尾的下一个位置(队满：rear == MAXSIZE),否则data[rear] = 入队元素，++rear

出队： 如果队列空则不能出队
队列空： front == rear 出队：++front,出队元素 == data[front]

ps:如果front指向队首，rear指向队尾的下一个位置,队空：front == rear， 出队：出队元素 == data[front], front++

读取队头元素：队头元素 == data[rear]

ps:如果front指向队首，rear指向队尾的下一个位置,队头元素 == data[rear-1]

顺序队列的缺点：队尾溢出，但是实际上数组仍有空间，从而引出了循环队列

循环队列

为了解决队尾溢出而实际上数组仍然有空余空间的问题，一般在队列的顺序存储结构中采用循环队列的方式:rear和front到达数组端点时，能折回到数组开始处，即相当于将数组头尾相接，想象成环状，如图所示。当插人和删除操作的作用单元达到数组的末端后,用公式“Rear(或Front)%数组长度”取余运算就可以实现折返到起始单元

判断队满还是对空的3种方法：

  1. 牺牲一个单元来区分队空和队满，入队时少用一个队列单元(上图的d2) 选用
  2. 类型中增设表示元素个数的数据成员.对空为size==0,队满为size==MAXSIZE，这两种情况都有front == rear
  3. 类型中增设tag成员，出队是tag==0，若因出队导致则为队空；入队时tag == 1，若因入队导致，则为队满

初始化：front = rear = 0

入队：先判断是否满 不满则pQ->rear = (pQ->rear + 1) % MAXSIZE、pQ->data[pQ->rear] = 入队元素

出队：先判断是否空 不空则pQ->front = (pQ->front + 1) % MAXSIZE 、p出队元素 = pQ->data[pQ->front]

队列元素的个数：(pQ->rear - pQ->front + MAXSIZE) % MAXSIZE

ps: 只能使用数组长度-1个空间，而且长度事先分配好(可以使用realloc扩增，但是也要消耗时间)
